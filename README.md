# Assignment 4. Опрацювання великих обсягів даних. Від сирих файлів до аналітики

## Мета
Навчитися зчитувати сирі бізнес-дані з файлів формату CSV та JSON, проводити їх очищення, зв'язувати дані з різних джерел за допомогою словників мови Python, розраховувати економічні показники та зберігати оновлені масиви даних. Продемонструвати вміння перетворювати словники у DataFrame для зручного перегляду та візуалізувати отримані бізнес-інсайти.

Для фахівця в галузі економіки та великих даних (EBD) програмування — це не про створення складних алгоритмів, а про **швидкість та автоматизацію**. Вміння завантажити тисячі рядків продажів, миттєво відфільтрувати помилки ("N/A"), розрахувати чистий прибуток та знайти найуспішніші категорії товарів за допомогою базових словників (dictionaries) зробить вашу роботу ефективною та безпомилковою.

## Завдання
**Контекст:** Ви працюєте аналітиком даних у міжнародній ритейл-мережі. У вас є два сирих файли від різних департаментів, які потрібно об'єднати, очистити та проаналізувати.

1. Зробіть форк репозиторію викладача (https://github.com/yKozolup/EBD_CS201_Assignment4) та склонуйте його, щоб отримати доступ до файлів:
   * `global_sales.csv` — база продажів (колонки: `transaction_id`, `date`, `region`, `product_category`, `quantity`, `revenue`). 
   * `regional_tariffs.json` — словник операційних витрат/тарифів у відсотках для кожного регіону.
2. Зчитайте дані з CSV у список словників (`list of dictionaries`), а дані з JSON — у звичайний словник (`dictionary`).
3. **Очищення даних (Data Cleaning):** У сирих базах часто бувають пропуски. Пройдіться по ваших даних. Якщо у колонках `quantity`, `revenue` (у CSV) або у тарифах (в JSON) зустрічається значення `"N/A"`, замініть його на числовий `0`. Переведіть усі числа в числовий тип (`float` або `int`).
4. **Оновлення даних:** Пройдіться по списку продажів і розрахуйте для кожної транзакції нову колонку `net_profit` (чистий прибуток).
   * *Формула:* `net_profit = revenue - (revenue * (tariff / 100))`
   * *Увага:* Тариф беріть зі зчитаного JSON-файлу за ключем `region`.
5. Збережіть цей *оновлений* масив транзакцій (з новою колонкою) у новий файл `cleaned_sales_updated.csv`.
6. **Аналітика (Новий масив даних):** Створіть новий порожній словник. Пройдіться по оновлених продажах і підрахуйте **сумарний чистий прибуток** (`net_profit`) для кожної категорії товарів (`product_category`). 
7. Розрахуйте середній прибуток серед усіх категорій. Використовуючи базову функцію `filter` (або `list comprehension`), залиште лише ті категорії, прибуток яких **більший за середній**.
8. Відсортуйте успішні категорії за спаданням прибутку (використайте функцію `sorted` або метод `sort`).
9. Збережіть цей принципово новий, агрегований набір даних у файл `top_categories.json`.
10. **Візуалізація:** Передайте ваш фінальний відфільтрований словник топ-категорій у Pandas DataFrame (лише для гарного виведення на екран) та виведіть цю таблицю.
11. Використовуючи Matplotlib, побудуйте базовий `bar chart` (стовпчасту діаграму), яка відображає прибуток цих топ-категорій. Додайте заголовок, підписи осей та зробіть графік приємним для сприйняття.
12. Створіть Pull Request з вашого форку до оригінального репозиторію викладача та надішліть посилання на нього.

*Обмеження:* Основна логіка очищення, розрахунку та агрегації повинна бути виконана **виключно** за допомогою базових інструментів Python (цикли, словники, `list comprehension`, `filter`, `sorted`). Pandas дозволено використовувати тільки в п.10 для виведення готової таблиці.
